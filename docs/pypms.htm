PyPMS - PMS extensions written in python

Usage

  1. Double-click on the python file pypms.py. You will be prompted for a start date, and an end date. Enter dates in the form YYYYMMDD (e.g. 20050401 for 1 April 2005.  All dates are inclusive.
  2. You will be told the directory where the output will be placed. This is in a directory derived from the first day of the reporting period. Hereinafter, this directory will be referred to as $OUTDIR.
  3. The program will perform it's computations, which takes only a few minutes. When they are finished, you are advised of this fact. Pressing any key will cause the console to disappear.


Any fatal errors that are encountered are printed to the console.

Output

Output is sent to $OUTDIR, as described in section Usage.  There are 3 types of output:

  1. client-targetted timesheets - these are pdfs of the form CODE-EE.pdf , where CODE is the job code, EE is the employee initials. They are in a form suitable for submission to clients. It is not expected that all of these files will be of interest to clients.
  2. people.txt - a summary of all of the employees time allocation for the period. Thiis can be used by the secretary to cross-check overtime allowances.
  3. intermediary and internal information files - anything other than the above. There may exist a file called exception.txt, whose existence indicates that a fatal error occurred in the processing, and whose contents give a traceback of the error encountered. Only a developer is expected to inderstand the output.
